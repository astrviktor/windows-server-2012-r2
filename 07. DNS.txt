DNS - важная часть в AD DC, без DNS работать не будет.

При установке AD DC DNS ставится автоматически, и больше мы в нем после установки пока ничего не меняли.

В обычном DNS есть зоны Primary (rw) и Secondary (read only)
В AD зоны AD-Integrated (rw)

AD-DNS синхронизируется с помощью средств Active Directory

Основные виды записей DNS
	NS(Authoritative name server) указывает на DNS-серверы, которые отвечают за хранение остальных ресурсных записей домена. Количество NS записей должно строго соответствовать количеству всех обслуживающих его серверов.

	CNAME (Canonical name) — запись, которая отвечает за привязку поддоменов (например, www.site.ru) к каноническому имени домена (site.ru)

	A (address ipv4) нужна для связи домена с IP-адресом сервера.

	MX(Mail Exchanger) - Адрес почтового шлюза для домена. Состоит из двух частей — приоритета (чем число больше, тем ниже приоритет), и адреса узла

	SOA (Start of Authority) — начальная запись зоны, которая указывает, на каком сервере хранится эталонная информация о доменном имени.

	TXT(Text string) — запись, которая содержит любую текстовую информацию о домене. Для различных целей

	PTR - обратная DNS-запись, которая связывает IP-адрес сервера с его каноническим именем (доменом). PTR-запись применяется для фильтрации почты.

	И другие

	SRV (Service Location) - записи для AD с указанием расположения разных сервисов, необходимых для работы AD

FORWARDERS - если DNS сам не может разрешить запрос, обращается сюда.
ROOT HINTS - если FORWARDERS не помог, обращается сюда.

Полезные команды:
	nslookup *name* - разрешить *name* в IP
	nslookup *name* *dns* - разрешить *name* в IP через *dns*

	ipconfig /all - посмотреть DNS - сервера
	ipconfig /displaydns - посмотреть кэш DNS
	ipconfig /flushdns - очистить кэш DNS
